== Introduction

The DocOps Todo Extension allows you to create visually appealing todo list visualizations in your AsciiDoctor documents. Todo lists are useful for project management, task tracking, and organizing work items in a visual format.

This guide will help you understand how to create different styles of todo visualizations, from simple lists to advanced Kanban boards.

== Basic Concepts

=== What are Todo Visualizations?

Todo visualizations in the DocOps extension are SVG-based visual elements that display task lists with:

* Task descriptions
* Status tracking (Pending, In Progress, Completed, etc.)
* Priority levels (Low, Medium, High, Critical)
* Optional categories, assignees, and due dates
* Multiple layout options (Kanban, List, Grid, Timeline)

=== Todo Components

A todo visualization includes:

* **Tasks**: The main items to be completed
* **Status**: Current state of each task
* **Priority**: Importance level of each task
* **Categories**: Optional grouping for tasks
* **Assignees**: Who is responsible for each task
* **Due Dates**: When tasks should be completed

== AsciiDoctor Syntax for Todo Lists

To include todo lists in your AsciiDoctor document, you use the DocOps macro. The extension supports both table format and JSON format.

=== Table Format (Simple)

Here's the basic table format:

[source,asciidoc]
....
[docops,todo,controls=true]
----
title: My Project Tasks
layout=kanban
groupBy=status
---
Task | Status | Priority | Category | Assignee | Due Date | Tags | Description
Setup development environment | pending | high | Development | Alice | 2024-01-15 | setup,env | Configure local development tools
Create project wireframes | in_progress | medium | Design | Bob | 2024-01-20 | design,ui | Design initial user interface mockups
Implement user authentication | pending | high | Development | Alice | 2024-01-25 | auth,backend | Add login and registration functionality
Write API documentation | pending | medium | Documentation | Carol | 2024-01-30 | docs,api | Document REST API endpoints
Deploy to staging | pending | low | DevOps | Bob | 2024-02-05 | deploy,staging | Set up staging environment
----
....

=== JSON Format (Advanced)

For more control, you can use the JSON format:

[source,asciidoc]
....
[docops,todo,controls=true]
----
{
  "config": {
    "title": "Sprint Planning",
    "layout": "KANBAN",
    "groupBy": "STATUS",
    "showStats": true,
    "showCategories": true,
    "showAssignees": true,
    "showDueDates": true,
    "useGlass": true
  },
  "items": [
    {
      "task": "Design user dashboard",
      "status": "IN_PROGRESS",
      "priority": "HIGH",
      "category": "Frontend",
      "assignee": "Alice",
      "dueDate": "2024-01-20",
      "tags": ["ui", "dashboard"],
      "description": "Create responsive dashboard layout"
    },
    {
      "task": "API integration tests",
      "status": "PENDING",
      "priority": "MEDIUM",
      "category": "Backend",
      "assignee": "Bob",
      "dueDate": "2024-01-25",
      "tags": ["testing", "api"],
      "description": "Write comprehensive API tests"
    }
  ]
}
----
....

== Configuration Options

=== Layout Types

* **KANBAN**: Board with columns for each status (default)
* **LIST**: Simple vertical list of tasks
* **GRID**: Grid layout with task cards
* **TIMELINE**: Timeline view (best with due dates)

=== Status Options

* **PENDING** (‚è≥): Not started yet
* **IN_PROGRESS** (üîÑ): Currently being worked on
* **COMPLETED** (‚úÖ): Finished
* **BLOCKED** (üö´): Cannot proceed
* **CANCELLED** (‚ùå): No longer needed
* **ON_HOLD** (‚è∏Ô∏è): Temporarily paused

=== Priority Levels

* **LOW** (üîµ): Nice to have
* **MEDIUM** (üü°): Normal priority
* **HIGH** (üü†): Important
* **CRITICAL** (üî¥): Urgent

=== Grouping Options

* **NONE**: No grouping
* **STATUS**: Group by task status
* **PRIORITY**: Group by priority level
* **CATEGORY**: Group by category
* **ASSIGNEE**: Group by assignee

== Best Practices

1. **Keep task names concise** - Use clear, actionable language
2. **Use consistent categories** - This helps with organization and filtering
3. **Set realistic priorities** - Don't make everything critical
4. **Regular updates** - Keep status current for team visibility
5. **Use due dates wisely** - Only when truly needed
6. **Leverage tags** - For cross-cutting concerns and filtering
