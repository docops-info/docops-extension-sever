== Introduction

_Task tracking with Kanban boards, lists, and grids_

*Key features:*

* Multiple layouts (Kanban, List, Grid, Timeline)
* Status tracking (Pending, In Progress, Completed, Blocked)
* Priority levels with visual indicators
* Assignee and due date support
* Table or JSON format

* *Project management* - Track deliverables across teams
* *Documentation tasks* - Organize writing and review assignments
* *Personal productivity* - Individual task lists with priorities

== Format Options

=== Table Format (Simple)
[source,asciidoc]
....
[docops,todo]
----
title: My Tasks
layout=kanban
groupBy=status
---
Task | Status | Priority | Category | Assignee | Due Date
Task name | pending | high | Category | Person | 2024-01-15
----
....

=== JSON Format (Advanced)
[source,asciidoc]
....
[docops,todo]
----
{
  "config": {
    "title": "Sprint Tasks",
    "layout": "KANBAN",
    "groupBy": "STATUS",
    "showStats": true
  },
  "items": [
    {
      "task": "Design dashboard",
      "status": "IN_PROGRESS",
      "priority": "HIGH",
      "category": "Frontend",
      "assignee": "Alice",
      "dueDate": "2024-01-20"
    }
  ]
}
----
....

[cols="1,2"]
|===
|Option |Purpose

|layout
|"kanban", "list", "grid", "timeline"

|groupBy
|"status", "priority", "category", "assignee", "none"

|showStats
|Display task count statistics

|showCategories
|Show category labels

|showAssignees
|Display assignee names

|showDueDates
|Show due dates

|useDark=true
|Dark theme
|===

== Best Practices

* *Keep task names actionable* - "Deploy API" not "API stuff"
* *Use consistent categories* - Helps with groupBy organization
* *Set priorities realistically* - Not everything is critical
* *Update status regularly* - Stale boards lose value quickly
* *Limit due dates* - Only when truly needed for coordination
* *Choose layout by context* - Kanban for workflow, List for quick reference

[TIP]
====
*For sprint planning:* Use Kanban layout grouped by status. Set HIGH priority for sprint commitments and MEDIUM for stretch goals. Update status daily during standup to keep the board accurate and actionable.
====
