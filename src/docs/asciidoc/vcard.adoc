== Introduction
The VCard Maker renders a clean, scannable contact card (SVG) from standard vCard 3.0 text. Itâ€™s great for team directories, speaker bios, internal wikis, and handoff documents. You write a tiny vCard block; DocOps turns it into a polished card with name, role, organization, phones, emails, and a compact QR code linking back to the card data.

== Usage
Insert a block using the docops extension and the vcard type. For reusability, keep your vCard content in data files and include them.

[source,asciidoc]
....
[docops,vcard]
----
include::data/vcard/vcard1-basic.txt[]
----
....

== Designs
The VCard Maker supports multiple design templates and a light/dark theme. Choose a design via a small config header placed before your vCard content, separated by ---.

Supported designs:

- tech_pattern_background â€” Tech motif background with contrast blocks; pairs well with theme=dark
- business_card_design â€” Classic business look (default if not specified)
- business_card_design2 â€” Modern variant of the classic business card
- business_card_template â€” Minimal template with clear information areas
- creative_agency_pro_contact_card â€” Playful, agency-style layout with bold accents
- modern_card â€” Contemporary, airy layout with generous spacing

Themes:

== Attributes

- design: Optional; selects the visual template. One of:
* tech_pattern_background,
* business_card_design,
* business_card_design2,
* business_card_template,
* creative_agency_pro_contact_card,
* modern_card.
Defaults to business_card_design when omitted or invalid.
- theme: Optional; light or dark. Defaults to light.
- scale: Optional numeric factor to scale the rendered SVG. Defaults to 1.0.

Notes on styling:
- Colors, typography, and layout follow the chosen design and theme. Designs may emphasize different typography, spacing, or accents, while theme adjusts contrast.

== Tips for great cards
- Prefer FN (Formatted Name) and N (structured name) together so names render crisply and sort correctly.
- Keep TITLE concise (e.g., Principal Engineer vs Principal Engineer â€” Platform Reliability) and move details to NOTE.
- Include at least one EMAIL or TEL; the card will adapt gracefully if one is missing.
- Use URL to link a profile or docs page; the QR encodes the vCard so mobile users can save the contact quickly.

== Notes
[NOTE]
====
- The input format is vCard 3.0 lines inside the block body. The maker supports FN, N, ORG, TITLE, TEL, EMAIL, URL, NOTE and gracefully ignores unknown properties.
- For maintainability, keep vCard blocks in data files and include them, so the same card can be reused across adoc pages.
