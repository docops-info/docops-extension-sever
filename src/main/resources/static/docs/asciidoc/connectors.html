1. Introduction

Connectors visualize sequential processes with labeled boxes and arrows. Perfect for workflows, decision flows, or any step-by-step procedure where order matters and each step needs a description.

1.1. Key Features

  • Auto-lettered boxes - Steps labeled A, B, C automatically

  • Arrow connections - Visual flow from left to right

  • Color-coded steps - Assign colors by phase or importance

  • Legend support - Descriptions appear below the flow

  • Dark mode optimized - Clean appearance in both themes

6.2. JSON Format

{
  "connectors": [
    {
      "text": "First Step",
      "baseColor": "#E14D2A",
      "description": "Description text"
    }
  ]
}

6.3. Common Options

  • useDark=true - Enable dark theme

  • Column order: text (step name), baseColor (hex), description

7. Best Practices

  • Limit to 5-7 steps - More steps become hard to follow visually

  • Color by phase - Use different colors for distinct phases (setup, process, output)

  • Keep step names short - 2-3 words; details go in description

  • Use consistent colors - Same color for similar step types across diagrams

  • Add descriptions - Legend helps users understand each step’s purpose

For deployment pipelines, use traffic light colors: green for automated steps, yellow for manual approvals, red for rollback triggers. This creates an instant visual understanding of risk and automation levels.
