1. Bar Charts

1.1. Introduction

Bar charts turn category comparisons into visual rankings. Whether you’re tracking sales by region, comparing quarterly performance, or visualizing survey results, bar charts make differences immediately scannable.

1.1.1. Key Features

  • Vertical & horizontal layouts - Choose the orientation that fits your data

  • Grouped comparisons - Show multiple series side-by-side

  • Cylinder style - 3D effect for presentations

  • Auto-sorting - Rank by value for clearer insights

  • Dark mode support - Optimized for both light and dark themes

1.6.2. Grouped Bar Chart

[docops,bargroup]
----

Group 1 | Category 2 | 7000.0
Group 2 | Category 1 | 6000.0

1.6.3. Common Options

  • type - R (regular), C (cylinder)

  • vBar=true - Vertical bars (default: horizontal)

  • sorted=true - Sort bars by value

  • useDark=true - Enable dark theme

  • scale=1.5 - Increase chart size

  • paletteType - OCEAN_BREEZE, CORPORATE, SUNSET (for grouped charts)

1.6.4. Custom Colors

Add color as third column:

Category 1 | 120.0 | #3498db
Category 2 | 334.0 | #e74c3c

ðŸ”

1.7. Best Practices

  • Use horizontal bars for long labels - Easier to read category names

  • Sort by value for rankings - Add sorted=true to highlight top/bottom performers

  • Group related series - Use bargroup for before/after or plan vs. actual comparisons

  • Limit groups to 3-5 - More than 5 series creates visual clutter

  • Use cylinder style sparingly - Best for high-level executive summaries, not detailed analysis

For time-series data (monthly trends), use vertical bars (vBar=true) with months on the X-axis. For category comparisons (regions, products), use horizontal bars with sorted values for instant ranking.

2. Line Charts

2.1. Introduction

Line charts reveal trends over time. Whether tracking daily metrics, monthly performance, or quarterly growth, line charts make patterns, peaks, and valleys immediately visible. Multiple series let you compare related trends side-by-side.

2.1.1. Key Features

  • Multi-series support - Compare up to 6 trends on one chart

  • Smooth or angular lines - Choose the visual style that fits your data

  • Auto-scaled axes - Smart axis ranges based on your data

  • Interactive legend - Toggle series visibility

  • Area fill option - Emphasize volume under the curve

2.6.2. Common Options

  • smoothLines=true - Curved lines (default: false for angular)

  • showArea=true - Fill area under lines (default: true)

  • useDark=true - Dark theme

  • width=800 - Chart width (default: 900)

  • backgroundColor=#F5F5F5 - Custom background color

ðŸ”

2.7. Best Practices

  • Use consistent time intervals - Don’t skip months or mix weekly/monthly data

  • Limit to 4-6 series - More lines create visual confusion

  • Choose smooth lines for trends - Angular lines for exact data points

  • Start Y-axis at zero - Unless showing small variations in large numbers

  • Label key inflection points - Annotate major changes or milestones

For volatile data (daily stock prices), use smooth lines to show overall trend. For discrete measurements (monthly sales), use angular lines to represent actual data points accurately.

3. Pie Charts

3.1. Introduction

Pie charts show proportions of a whole at a glance. Perfect for market share, budget breakdowns, or any data where the parts sum to 100%. The modern donut style adds visual polish while maintaining clarity.

3.1.1. Key Features

  • Classic pie or modern donut - Choose the style that fits your aesthetic

  • Auto percentages - Labels show both values and calculated percentages

  • Interactive legend - Hover effects for engagement

  • Glass morphism design - Modern, polished appearance

  • Dark mode support - Seamless light/dark theme switching

3.7. Best Practices

  • Limit slices to 5-7 - More slices become hard to distinguish visually

  • Order by size - Place largest slice at 12 o’clock, descend clockwise

  • Use donut for modern docs - The donut style looks more polished and less dated

  • Show percentages for small slices - Values under 10% are hard to judge by area alone

  • Avoid pie charts for time series - Use line or bar charts for trends over time

If you have more than 7 categories, group smaller slices into an "Other" category. Pie charts lose effectiveness when slices become too thin to label clearly.

3.8.5. Best Practices

  • Limit to 4-6 pies - More pies become cluttered horizontally

  • Use semantic colors - Green outline for success, red for warnings

  • Keep labels short - 1-2 words for clean layout

  • Scale for context - 0.8 for cards, 1.2 for hero metrics

For dashboards: Use scale=0.9 with consistent baseColor/outlineColor across all metrics. This creates a unified visual language where users instantly recognize "green outline = healthy, red outline = attention needed."

4. Gauge Charts

4.1. Introduction

Gauge charts turn single metrics into visual speedometers, perfect for monitoring performance indicators, resource utilization, and progress tracking. They provide instant status feedback through color-coded ranges and smooth animations.

4.1.1. Key Features

  • 7 gauge types - From classic speedometers to modern digital displays

  • Auto color zones - Normal (green), caution (yellow), critical (red)

  • Animated arcs - Smooth transitions enhance readability

  • Dark mode support - Optimized for both light and dark themes

  • Flexible layouts - Single gauge or multi-gauge grids

4.6.2. Common Options

  • useDark=true - Enable dark theme

  • showRanges=true - Display color zones (default: true)

  • showArc=true - Show arc (for DIGITAL type)

  • animateArc=true - Animate the arc (default: true)

  • columns=3 - Number of columns for MULTI_GAUGE

4.7. Color Ranges

Gauges automatically apply color zones:

  • 0-60% - Normal (Green #10B981) - Optimal performance

  • 60-85% - Caution (Yellow #F59E0B) - Warning zone

  • 85-100% - Critical (Red #EF4444) - Action required

Disable with showRanges=false to use custom colors.

ðŸ”

4.8. Common Use Cases

  • Infrastructure monitoring - CPU, memory, disk usage across server fleets

  • Progress tracking - Project milestones, sprint burndown, goal completion

  • Environmental metrics - Temperature, humidity, air quality in facilities

  • Budget dashboards - Spend vs. allocation, runway visualization

  • Service health - Uptime percentages, response times, error rates

ðŸ”

4.9. Best Practices

  • Use MULTI_GAUGE for comparisons - Show 3-9 related metrics side-by-side

  • Reserve DIGITAL for hero metrics - One prominent number that demands attention

  • Set realistic ranges - Min/max should reflect actual operating bounds

  • Add targets for LINEAR - Show goal lines for budget or performance targets

  • Keep units consistent - Don’t mix % and absolute values in one grid

For server monitoring, use MULTI_GAUGE with 3 columns. This creates a scannable grid where anomalies (red/yellow gauges) pop instantly.

5. Combination Charts

5.1. Introduction

Combination charts layer bars and lines on one chart, perfect for showing related metrics with different scales. Track revenue (bars) against growth rate (line), or sales volume (bars) versus profit margin (line). Dual Y-axes keep everything readable.

5.1.1. Key Features

  • Mix bars and lines - Combine chart types for richer insights

  • Dual Y-axes - Show metrics with different scales side-by-side

  • Smart legends - Shows chart type for each series

  • Smooth or angular lines - Visual control over line appearance

  • Auto-scaling - Each Y-axis scales independently

5.5. Format Options

5.5.1. Basic Format

[docops,combination]
----
title=Chart Title
xLabel=X-Axis Label
yLabel=Primary Y-Axis Label
yLabelSecondary=Secondary Y-Axis Label
dualYAxis=true
smoothLines=true
showGrid=true
---
Series Name | Type | X-Value | Y-Value | Color | Y-Axis
Revenue | BAR | Q1 | 50000 | #3498db | PRIMARY
Growth Rate | LINE | Q1 | 15.5 | #e74c3c | SECONDARY
----

5.5.2. Common Options

  • dualYAxis=true - Enable second Y-axis for different scales

  • smoothLines=true - Curved lines (default: false)

  • showGrid=true - Display background grid (default: true)

  • showLegend=true - Display legend (default: true)

  • showPoints=true - Show data point markers on lines (default: true)

  • useDark=true - Dark theme

5.5.3. Data Columns

  • Series Name - Label shown in legend

  • Type - BAR or LINE

  • X-Value - Category or time period label

  • Y-Value - Numeric value

  • Color - Hex color (e.g., #3498db)

  • Y-Axis - PRIMARY or SECONDARY

ðŸ”

5.6. Best Practices

  • Use dual Y-axes for different units - Revenue ($) on left, rate (%) on right

  • Bars for absolutes, lines for rates - Volume as bars, efficiency as lines

  • Match colors to meaning - Blue for revenue, red for costs, green for profit

  • Limit to 2-4 series - One bar series, 1-3 line series maximum

  • Label both Y-axes clearly - Include units in axis labels

Combination charts excel when one metric drives the other (e.g., marketing spend drives conversions). Use bars for the driver (input) and lines for the outcome (result) to show cause-and-effect visually.
