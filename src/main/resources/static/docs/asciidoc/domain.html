2. Introduction

Domain visualizations map business or technical architectures using a compact CSV format that generates clean, hierarchical diagrams. Perfect for showing system landscapes, capability maps, or any structure with shared foundations and specialized verticals.

2.1. Key Features

  • Hierarchical layout - Main node, common capabilities, specialized groups

  • Emoji support - Visual icons for quick identification

  • Clickable nodes - Wiki-style links embed documentation

  • Glass morphism* - Modern, layered visual design

  • Auto-layout - Smart positioning prevents overlaps

7.2. Row Organization

  • COMMON rows - Shared capabilities across all verticals (appear at top)

  • Specialized groups - Named verticals with their own rows

  • rowIndex - Order within a group (0=first row, 1=second row, etc.)

  • nodes - Comma-separated list (quoted if contains commas)

7.3. Wiki-Style Links

Transform nodes into clickable links:

"[[https://docs.com API Docs]],SERVICE,DATABASE"

Multiple links per node (semicolon-separated):

"[[https://github.com/repo Code]];[[https://wiki.com Docs]]"

7.4. Common Options

  • useDark=true - Enable dark theme

  • useNeural=true - Alternative neural network visual style

8. Best Practices

  • Start with main node - Always declare main,NAME first

  • Group by purpose - Use COMMON for shared, named groups for verticals

  • Limit rows per group - 2-3 rows max for readability

  • Use emojis sparingly - One per vertical for visual distinction

  • Link to documentation - Make diagrams navigable knowledge hubs

  • Keep node names short - 1-2 words max; details go in linked docs

For architecture diagrams, use COMMON rows for infrastructure (databases, auth, monitoring,) and specialized groups for product verticals (web app, mobile app, API). This instantly shows what’s shared vs. what’s unique.
