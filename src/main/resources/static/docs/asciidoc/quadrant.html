1. Introduction

2D priority mapping for strategic decision-making

Key features:

  • Four-quadrant grid for impact vs. effort analysis

  • Customizable axis and quadrant labels

  • Category-based color coding for data points

  • Visual Version 2 with enhanced design

  • Dark mode support for presentations

7. Data Format

Each data point has 3-4 pipe-separated values:

Label | X | Y [| Category]

  • Label (required) - Item name displayed on chart

  • X (required) - Horizontal position (0-100)

  • Y (required) - Vertical position (0-100)

  • Category (optional) - Groups items by color

8. Best Practices

  • Label quadrants contextually - "Quick Wins" vs. "Big Bets" is clearer than generic "High/Low"

  • Limit to 8-12 items - Too many points create visual clutter

  • Use categories for grouping - Color-code by team, phase, or priority tier

  • Position strategically - Place similar items apart to avoid label overlap

  • Match axes to analysis - Flip axes (complexity vs. value) when it clarifies meaning

  • Add dark mode for stakeholders - Professional look for executive presentations

For risk assessment: Place X-axis as "Probability" and Y-axis as "Impact". This matches industry standard risk matrices where top-right items (high probability + high impact) demand immediate attention.
