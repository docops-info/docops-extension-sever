Bar Charts

1. Introduction

Bar charts turn category comparisons into visual rankings. Whether you’re tracking sales by region, comparing quarterly performance, or visualizing survey results, bar charts make differences immediately scannable.

1.1. Key Features

  • Vertical & horizontal layouts - Choose the orientation that fits your data

  • Grouped comparisons - Show multiple series side-by-side

  • Cylinder style - 3D effect for presentations

  • Auto-sorting - Rank by value for clearer insights

  • Dark mode support - Optimized for both light and dark themes

🔝

3. Common Use Cases

  • Sales dashboards - Compare revenue across time periods or regions

  • Survey results - Show response distributions and rankings

  • Performance metrics - Track KPIs against targets or benchmarks

  • Resource allocation - Visualize budget or headcount by department

  • A/B test results - Compare variant performance side-by-side

🔝

4. Format Options

4.1. Simple Bar Chart

[docops,bar]
----
title=Chart Title
yLabel=Y-Axis Label
xLabel=X-Axis Label
type=R
vBar=false
sorted=true
---
Category 1 | 120.0
Category 2 | 334.0
Category 3 | 455.0
----

4.2. Grouped Bar Chart

[docops,bargroup]
----
title=Chart Title
yLabel=Y-Axis Label
xLabel=X-Axis Label
paletteType=OCEAN_BREEZE
vBar=false
---
Group 1 | Category 1 | 5000.0
Group 1 | Category 2 | 7000.0
Group 2 | Category 1 | 6000.0
Group 2 | Category 2 | 8000.0
----

4.3. Common Options

  • type - R (regular), C (cylinder)

  • vBar=true - Vertical bars (default: horizontal)

  • sorted=true - Sort bars by value

  • useDark=true - Enable dark theme

  • scale=1.5 - Increase chart size

  • paletteType - OCEAN_BREEZE, CORPORATE, SUNSET (for grouped charts)

4.4. Custom Colors

Add color as third column:

Category 1 | 120.0 | #3498db
Category 2 | 334.0 | #e74c3c

🔝

5. Best Practices

  • Use horizontal bars for long labels - Easier to read category names

  • Sort by value for rankings - Add sorted=true to highlight top/bottom performers

  • Group related series - Use bargroup for before/after or plan vs. actual comparisons

  • Limit groups to 3-5 - More than 5 series creates visual clutter

  • Use cylinder style sparingly - Best for high-level executive summaries, not detailed analysis

For time-series data (monthly trends), use vertical bars (vBar=true) with months on the X-axis. For category comparisons (regions, products), use horizontal bars with sorted values for instant ranking.
