== Common Use Cases

* *Feature prioritization* - Map complexity vs. business value for roadmap planning
* *Risk management* - Plot probability vs. impact for risk mitigation strategies
* *Technology evaluation* - Gartner-style Magic Quadrant for vendor comparisons
* *Portfolio analysis* - Investment opportunities by risk vs. return
* *Sprint planning* - Effort estimation vs. stakeholder value for backlog grooming

== Format Options

=== Basic Format
[source,asciidoc]
....
[docops,quadrant]
----
title=Chart Title
xAxis= X-Axis Label
yAxis= Y-Axis Label
---
Item Name | X | Y | Category
----
....

=== Custom Quadrants
[source,asciidoc]
....
[docops,quadrant]
----
title= Custom Labels
leaders= TOP RIGHT
challengers= TOP LEFT
visionaries= BOTTOM LEFT
niche= BOTTOM RIGHT
---
Item | 50 | 50 | Category
----
....

=== Visual Version 2
[source,asciidoc]
....
[docops,quadrant,useDark=true]
----
visualVersion=2
title= Modern Design
---
Item | X | Y | Category
----
....

|===

|X-axis label (default: "EFFORT REQUIRED")

|Y-axis label (default: "IMPACT LEVEL")

|Bottom-left quadrant (default: "FILL-INS")

|niche
|Bottom-right quadrant (default: "THANKLESS")

|visualVersion=2
|Enhanced visual design

== Quadrant Positioning

[cols="1,1,3"]
|===
|Quadrant |Position |Typical Meaning

|Leaders
|Top-right (high Y, high X)
|High impact, high effort (big bets)

|Challengers
|Top-left (high Y, low X)
|High impact, low effort (quick wins)

|Visionaries
|Bottom-left (low Y, low X)
|Low impact, low effort (fill-ins)

|Niche
|Bottom-right (low Y, high X)
|Low impact, high effort (avoid)
|===

== Data Format

Each data point has 3-4 pipe-separated values:

[source]
----
Label | X | Y [| Category]
----

* *Label* (required) - Item name displayed on chart
* *X* (required) - Horizontal position (0-100)
* *Y* (required) - Vertical position (0-100)
* *Category* (optional) - Groups items by color

== Best Practices

* *Label quadrants contextually* - "Quick Wins" vs. "Big Bets" is clearer than generic "High/Low"
* *Limit to 8-12 items* - Too many points create visual clutter
* *Use categories for grouping* - Color-code by team, phase, or priority tier
* *Position strategically* - Place similar items apart to avoid label overlap
* *Match axes to analysis* - Flip axes (complexity vs. value) when it clarifies meaning
* *Add dark mode for stakeholders* - Professional look for executive presentations

[TIP]
====
*For risk assessment:* Place X-axis as "Probability" and Y-axis as "Impact". This matches industry standard risk matrices where top-right items (high probability + high impact) demand immediate attention.
====
