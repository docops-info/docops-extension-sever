== Introduction

Buttons create organized navigation grids that turn link lists into scannable visual menus. Perfect for dashboards, index pages, and resource directories where you need users to find the right link fast.

=== Key Features

* *9 button shapes* - From regular rectangles to hexagons and circles
* *Auto color-coding* - Assign colors by type for visual grouping
* *Flexible layouts* - Control columns, scale, and spacing
* *Embedded images* - Add icons or logos to buttons
* *Sorting options* - Order by label, type, date, or custom order

== Button Shape Reference

[cols="1,3"]
|===
| Button Type | Best For

| `REGULAR`
| Standard navigation grids, link directories

| `PILL`
| Modern UI, tag-style navigation

| `LARGE`
| Hero sections, primary actions (2-4 buttons max)

| `RECTANGLE`
| Documentation indexes, structured menus

| `ROUND`
| Dashboard widgets, feature showcases

| `CIRCLE`
| Icon-focused navigation, app launchers

| `SLIM`
| Compact menus, sidebar navigation

| `OVAL`
| Playful designs, creative projects

| `HEX`
| Technical docs, architecture diagrams
|===

== Common Use Cases

* *Documentation indexes* - Organize guides, tutorials, and references
* *Resource directories* - Link to tools, repos, and external docs
* *Dashboard navigation* - Quick access to different sections or apps
* *Feature showcases* - Highlight product capabilities with CTAs
* *Team directories* - Link to team pages or contact info

== Format Options

=== JSON Structure

[source,json]
....
{
  "buttons": [
    {
      "label": "Button Label",
      "link": "https://example.com",
      "description": "Optional description",
      "type": "category"
    }
  ],
  "buttonType": "REGULAR",
  "theme": {
    "colors": ["#003b6b", "#FF6F36"],
    "columns": 2,
    "scale": 1.0,
    "buttonStyle": {
      "labelStyle": "font-family: Arial; font-size: 12px; fill: #fcfcfc;"
    }
  }
}
....

=== Common Theme Options

* `buttonType` - REGULAR, PILL, LARGE, RECTANGLE, ROUND, CIRCLE, SLIM, OVAL, HEX
* `colors` - Array of hex colors for auto-assignment by type
* `columns` - Number of columns (2-4 recommended)
* `scale` - Size multiplier (default: 1.0)
* `useDark=true` - Dark Mode
* `newWin: true` - Open links in new window
* `sortBy.sort` - ORDER, LABEL, TYPE, DATE, AUTHOR


=== Embedded Images

Add icons to buttons:

[source,json]
....
{
  "label": "Documentation",
  "link": "https://docs.example.com",
  "embeddedImage": {
    "ref": "images/docs-icon.svg"
  }
}
....

== Best Practices

* *Use 2-3 columns* - More columns create cramped buttons on smaller screens
* *Color-code by type* - Assign consistent colors to categories (docs=blue, tools=green)
* *Keep labels short* - 1-3 words max for readability
* *Limit grid size* - 6-12 buttons per grid; more creates decision fatigue
* *Match shape to context* - Use REGULAR for general navigation, LARGE for primary CTAs

[TIP]
====
For documentation indexes, use type-based color coding (e.g., "tutorial"=blue, "reference"=green, "guide"=orange). Users quickly learn the color system and can scan faster.
====
