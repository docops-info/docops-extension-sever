1. Introduction

Visualize product release plans with timeline roadmaps

Key features:

  • Timeline (TLS) and Roadmap ® layouts for different visualization needs

  • Release types (M1, M2, RC1, GA) with dates, goals, and detailed activities

  • Modern glass design with translucent depth effects

  • Dark mode support with custom color palettes

  • Scalable visualizations for presentations and documents

4. Common Use Cases

  • Product roadmaps - Communicate quarterly/annual release plans to stakeholders

  • Sprint planning - Visualize agile development cycles with milestone markers

  • Migration projects - Track multi-phase infrastructure or platform transitions

  • Mobile app launches - Coordinate design, development, testing, and release phases

  • Compliance initiatives - Map audit preparation, testing, and certification timelines

5.2. Roadmap ® with Custom Colors

[docops,release]
----
{
  "title": "Annual Roadmap",
  "style": "R",
  "scale": 0.8,
  "useDark": true,
  "releases": [...],
  "displayConfig": {
    "fontColor": "#e2e8f0",
    "colors": ["#06B6D4", "#6366F1", "#10B981"],
    "notesVisible": true
  }
}
----
Option Purpose

style: "TLS"

Condensed timeline view

style: "R"

Detailed roadmap layout

useDark: true

Dark theme for presentations

scale: 0.7-1.5

Size adjustment for different contexts

displayConfig.colors

Custom color palette (array of 3 hex colors)

displayConfig.notesVisible

Toggle activity details visibility

6. Best Practices

  1. Use semantic release types - M1/M2 for milestones, RC1/RC2 for release candidates, GA for general availability

  2. Keep activity lists focused - 3-5 activities per release for readability

  3. Choose the right style - TLS for high-level overviews, R for detailed quarter-by-quarter plans

  4. Scale appropriately - Use scale: 1.5 for presentations, 0.7-0.8 for embedded documentation

  5. Leverage dark mode - Dark theme with cyan/purple/green palette creates professional look for stakeholder presentations

  6. Align dates consistently - Use ISO dates (YYYY-MM-DD) or quarters (YYYY-Q1) throughout

For multi-year roadmaps: Use the Roadmap ® style with quarterly dates (2023-Q1, 2023-Q2) rather than specific dates. This creates breathing room between releases and accommodates the glass design’s translucent depth effects without overcrowding.
