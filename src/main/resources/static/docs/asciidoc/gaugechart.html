1. Introduction

Gauge charts turn single metrics into visual speedometers, perfect for monitoring performance indicators, resource utilization, and progress tracking. They provide instant status feedback through color-coded ranges and smooth animations.

1.1. Key Features

  • 7 gauge types - From classic speedometers to modern digital displays

  • Auto color zones - Normal (green), caution (yellow), critical (red)

  • Animated arcs - Smooth transitions enhance readability

  • Dark mode support - Optimized for both light and dark themes

  • Flexible layouts - Single gauge or multi-gauge grids

6. Format Options

6.1. Basic Format

[docops,gauge]
----
type=SEMI_CIRCLE
title=Your Title
showRanges=true
---
Label | Value | Min | Max | Unit | Color
CPU Load | 72 | 0 | 100 | % |
----

6.2. Common Options

  • type - SEMI_CIRCLE, DIGITAL, MULTI_GAUGE, LINEAR, FULL_CIRCLE, SOLID_FILL, DASHBOARD

  • useDark=true - Enable dark theme

  • showRanges=true - Display color zones (default: true)

  • showArc=true - Show arc (for DIGITAL type)

  • animateArc=true - Animate the arc (default: true)

  • columns=3 - Number of columns for MULTI_GAUGE

6.3. Linear Gauge with Target

type=LINEAR
showTarget=true
---
Label | Value | Min | Max | Unit | Color | Target

7. Color Ranges

Gauges automatically apply color zones:

  • 0-60% - Normal (Green #10B981) - Optimal performance

  • 60-85% - Caution (Yellow #F59E0B) - Warning zone

  • 85-100% - Critical (Red #EF4444) - Action required

Disable with showRanges=false to use custom colors.

8. Common Use Cases

  • Infrastructure monitoring - CPU, memory, disk usage across server fleets

  • Progress tracking - Project milestones, sprint burndown, goal completion

  • Environmental metrics - Temperature, humidity, air quality in facilities

  • Budget dashboards - Spend vs. allocation, runway visualization

  • Service health - Uptime percentages, response times, error rates

9. Best Practices

  • Use MULTI_GAUGE for comparisons - Show 3-9 related metrics side-by-side

  • Reserve DIGITAL for hero metrics - One prominent number that demands attention

  • Set realistic ranges - Min/max should reflect actual operating bounds

  • Add targets for LINEAR - Show goal lines for budget or performance targets

  • Keep units consistent - Don’t mix % and absolute values in one grid

For server monitoring, use MULTI_GAUGE with 3 columns. This creates a scannable grid where anomalies (red/yellow gauges) pop instantly.
