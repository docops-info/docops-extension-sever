5. Common Use Cases

  • Sprint planning - Organize current sprint, backlog, and completed items

  • Product roadmaps - Show feature progression from now to future

  • Project phases - Break down project into stages with clear status

  • Team workflows - Visualize what each team is working on currently

  • Release planning - Track features across release milestones

6. Format Options

6.1. Basic Structure

- now Task Title
Task description goes here

- next Another Task
Another description

- later Future Task
Planning notes

- done Completed Task
What was accomplished

6.2. Common Options

  • title="Your Title" - Planner heading

  • useDark=true - Enable dark theme

6.3. Task Format

Each task starts with - stage (now, next, later, done) followed by: - Line 1: Task title - Line 2+: Task description (until next - stage)

7. Best Practices

  • Limit NOW to 2-4 items - Keep current focus small and achievable

  • NEXT for immediate queue - 3-5 items ready to start

  • LATER for backlog - Don’t overload; prioritize top items only

  • DONE for wins - Show recent completions to celebrate progress

  • One-line titles - Keep task titles concise; details in description

For sprint planning, use NOW for current sprint work, NEXT for sprint backlog, LATER for product backlog, and DONE for last sprint’s accomplishments. This creates a rolling window of team velocity.
