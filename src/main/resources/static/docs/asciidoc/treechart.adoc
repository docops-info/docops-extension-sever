== Introduction


_Hierarchical diagrams for org charts, taxonomies, and decision trees_

*Key features:*

* Vertical or horizontal tree layouts
* Interactive collapsible nodes for large hierarchies
* Custom color coding per node or palette-based
* Glass effect and dark mode support
* Indentation-based syntax for easy authoring

== Common Use Cases

* *Org charts* - Visualize reporting structures and team hierarchies
* *Decision trees* - Document branching logic for troubleshooting or workflows
* *Taxonomies* - Show classification systems for products, content, or data
* *Project breakdowns* - WBS (Work Breakdown Structure) for project planning
* *File system structures* - Illustrate directory hierarchies and dependencies

== Format Options

=== Basic Structure
[source,asciidoc]
....
[docops,treechart]
----
title=Chart Title
width=800
height=600
orientation=vertical
---
Root Node
    Child 1
        Grandchild 1
    Child 2
----
....

=== Custom Colors
[source,asciidoc]
....
// Per-node colors
----
Root | #3498db
    Child 1 | #2ecc71
    Child 2 | #e74c3c
----

// Palette-based colors
----
colors=#FF5733,#C70039,#900C3F,#581845
---
Root
    Child 1
    Child 2
----
....

[cols="1,2"]
|===
|Option |Purpose

|title
|Chart heading

|width / height
|Canvas dimensions (default: 800x600)

|orientation
|"vertical" (top-down) or "horizontal" (left-right)

|collapsible=false
|Disable interactive expand/collapse (for PDF)

|expanded=true
|Show all nodes initially

|useGlass=true
|Modern glass effect on nodes

|useDark=true
|Dark theme

|colors
|Comma-separated hex colors for palette
|===

== Data Syntax

Indentation defines hierarchy:

[source]
----
Root Node [| #color]
    Child Level 1
        Grandchild Level 2
            Great-grandchild Level 3
    Another Child Level 1
----

* Use **spaces or tabs** for indentation (consistent within document)
* Optional **| #hexcolor** suffix for custom node colors
* Root node must have **zero indentation**

== Best Practices

* *Keep depth to 4-5 levels* - Deep trees become hard to read
* *Use horizontal for wide trees* - Better for many branches at each level
* *Color-code by category* - Frontend (green), Backend (red), DevOps (purple)
* *Set collapsible=false for PDF* - Ensure all nodes visible in static exports
* *Increase dimensions for large trees* - width=1200 height=1000 for 20+ nodes
* *Use glass effect sparingly* - Better for feature roadmaps than dense org charts

[TIP]
====
*For org charts with 50+ people:* Use horizontal orientation with width=1600, height=1200. Color-code by department and set collapsible=true to let viewers drill down into specific teams interactively.
====
