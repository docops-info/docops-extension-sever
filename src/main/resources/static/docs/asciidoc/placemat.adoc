== Introduction

_Color-coded label grids for organizing concepts and components_

*Key features:*

* Grid-based layout with automatic wrapping
* Color-coded categories via legend system
* Filled or outlined box styles
* Dark mode for presentations
* JSON format for flexible configuration

== Common Use Cases

* *Technology inventories* - Catalog frameworks, libraries, and tools by category
* *Architecture diagrams* - Show system components grouped by layer or function
* *Glossaries* - Define and categorize domain terminology
* *Team skill matrices* - Display competencies organized by role or department
* *Compliance checklists* - Group requirements by regulatory framework

== Format Options

=== Basic Structure
[source,asciidoc]
....
[docops,placemat]
----
{
  "title": "Title Here",
  "placeMats": [
    {"name": "Item Name", "legend": "Category"}
  ],
  "config": {
    "legend": [
      {"legend": "Category", "color": "#4361ee"}
    ]
  }
}
----
....

=== Filled vs. Outlined
[source,asciidoc]
....
// Filled boxes (default)
[docops,placemat]
----
{
  "title": "Filled Style",
  "placeMats": [...],
  "config": {...}
}
----

// Outlined boxes
[docops,placemat]
----
{
  "title": "Outline Style",
  "fill": false,
  "placeMats": [...],
  "config": {...}
}
----
....

[cols="1,2"]
|===
|Option |Purpose

|title
|Diagram heading

|placeMats
|Array of {"name": "text", "legend": "category"}

|config.legend
|Array of {"legend": "category", "color": "#hex"}

|useDark=true
|Dark theme for presentations

|fill=false
|Outlined boxes instead of filled

|scale
|Size multiplier (default: 1.0)
|===

== JSON Structure

Each placemat item maps to a legend category:

[source,json]
----
{
  "placeMats": [
    {"name": "Component Name", "legend": "Category1"},
    {"name": "Another Component", "legend": "Category2"}
  ],
  "config": {
    "legend": [
      {"legend": "Category1", "color": "#4361ee"},
      {"legend": "Category2", "color": "#7209b7"}
    ]
  }
}
----

* *name* - Text displayed in box
* *legend* - Must match a legend category
* *color* - Hex color for category

== Best Practices

* *Limit categories to 5-7* - Too many colors reduce visual distinction
* *Use semantic colors* - Frontend (blue), Backend (purple), Database (dark purple) create natural associations
* *Keep names concise* - 1-3 words fit best in grid boxes
* *Group related items* - Order by category for visual clustering
* *Choose fill style contextually* - Filled for emphasis, outlined for subtle backgrounds
* *Combine with connectors* - Follow placemat with connector diagrams to show relationships

[TIP]
====
*For microservices architectures:* Create separate placemats for infrastructure layers (Gateway, Services, Databases). Then use connector visualizations to show data flows between specific components from each layer.
====
