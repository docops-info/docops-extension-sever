2. Introduction

Word clouds transform text data into visual hierarchies where size indicates importance. Perfect for survey results, tag clouds, keyword analysis, or any scenario where you want to show relative frequency or significance at a glance.

2.1. Key Features

  • Weight-based sizing - Higher weight = larger font

  • Custom colors - Assign colors per word or use palette

  • Auto-placement - Intelligent algorithm prevents overlaps

  • Interactive hover - Highlight words on mouseover

  • Responsive scaling - Scale entire cloud with one parameter

7. Format Options

7.1. Basic Format

[docops,wordcloud]
----
title= Your Title
minFontSize= 12
maxFontSize= 60
---
Word | Weight | Color
Term 1 | 95 | #3498db
Term 2 | 88 | #2ecc71
Term 3 | 72 | #e74c3c
----

7.2. Common Options

  • title - Cloud title (shown at top)

  • minFontSize=12 - Smallest font size (default: 10)

  • maxFontSize=60 - Largest font size (default: 60)

  • width=800 - Canvas width (default: 800)

  • height=600 - Canvas height (default: 600)

  • useDark=true - Enable dark theme

  • scale=1.5 - Scale entire cloud

7.3. Weight Guidelines

Weights determine relative sizing: - 90-100: Dominant terms (largest) - 70-89: Important terms (large) - 50-69: Common terms (medium) - 30-49: Supporting terms (small) - 10-29: Rare terms (smallest)

8. Best Practices

  • Use 15-30 words - Too few lacks impact; too many creates clutter

  • Weight by significance* - Base weights on frequency, importance, or impact

  • Color by category - Group related terms with similar colors

  • Leave white space - Don’t maximize font sizes; readability > density

  • Rotate for variety - Mix horizontal and angled words for visual interest

For survey results, weight by response frequency and color-code by sentiment (green=positive, red=negative, blue=neutral). This creates a visual sentiment map that’s instantly readable.
