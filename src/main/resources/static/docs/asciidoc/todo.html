6.2. JSON Format (Advanced)

[docops,todo]
----
{
  "config": {
    "title": "Sprint Tasks",
    "layout": "KANBAN",
    "groupBy": "STATUS",
    "showStats": true
  },
  "items": [
    {
      "task": "Design dashboard",
      "status": "IN_PROGRESS",
      "priority": "HIGH",
      "category": "Frontend",
      "assignee": "Alice",
      "dueDate": "2024-01-20"
    }
  ]
}
----
Option Purpose

layout

"kanban", "list", "grid", "timeline"

groupBy

"status", "priority", "category", "assignee", "none"

showStats

Display task count statistics

showCategories

Show category labels

showAssignees

Display assignee names

showDueDates

Show due dates

useDark=true

Dark theme

8. Best Practices

  • Keep task names actionable - "Deploy API" not "API stuff"

  • Use consistent categories - Helps with groupBy organization

  • Set priorities realistically - Not everything is critical

  • Update status regularly - Stale boards lose value quickly

  • Limit due dates - Only when truly needed for coordination

  • Choose layout by context - Kanban for workflow, List for quick reference

For sprint planning: Use Kanban layout grouped by status. Set HIGH priority for sprint commitments and MEDIUM for stretch goals. Update status daily during standup to keep the board accurate and actionable.
